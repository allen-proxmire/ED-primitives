# Architecture
### *The structural organization of the ED ontology*

The `architecture/` folder defines **how ED structure forms**, **how it stabilizes**, and **how it propagates**.  
It is not physics, not PDEs, not experiments — it is the **structural layer** of the ontology.

If the `primitives/` folder defines *what exists*,  
and the `foundations/` folder defines *why ED is built this way*,  
the `architecture/` folder defines *how structure emerges from the primitives*.

This is the **core structural engine** of ED.

---

# Purpose of This Folder

The Architecture layer answers questions like:

- How do gradients generate structure  
- How do sign‑flips produce saddles  
- How do saddles become boundaries  
- How do boundaries become horizons  
- How do motifs form and stabilize  
- How does structure propagate into networks  
- How does ED scale from local interactions to global architecture  

These documents describe the **forced structural pipeline** that arises whenever ED primitives interact.

# The Architectural Pipeline

All ED structure follows the same universal sequence:

```
Sign‑Flip → Saddle → Boundary → Horizon → Motif → Network
```

This pipeline is:

- **forced** by the primitives  
- **independent** of physics  
- **structural**, not dynamical  
- **geometric → topological → directional**  

Understanding this pipeline is the key to understanding ED.
