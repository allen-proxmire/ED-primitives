# conceptual_minimalism.md
### *Why ED uses the smallest possible ontology*

ED is intentionally minimal.  
It uses **exactly 13 primitives** — no more, no less.

This document explains why.

---

## 1. Minimalism prevents redundancy
If the ontology had more primitives:

- they would overlap  
- they would duplicate roles  
- they would obscure structure  

Thirteen is the **minimal complete set**.

---

## 2. Minimalism forces clarity
A small ontology:

- forces precision  
- forces clean architecture  
- forces explicit commitments  
- prevents conceptual drift  

ED is designed to be **crisp**, not sprawling.

---

## 3. Minimalism supports compositionality
ED relies on:

- motif formation  
- boundary reinforcement  
- horizon propagation  

These require **simple ingredients** that combine into **rich structure**.

---

## 4. Minimalism avoids metaphysical inflation
ED does not assume:

- substances  
- forces  
- fields  
- particles  
- hidden variables  
- extra dimensions  

Minimalism keeps ED **ontologically disciplined**.

---

## 5. Summary

ED is minimal because:

- fewer primitives collapse structure  
- more primitives add redundancy  
- exactly 13 primitives produce the full architecture  

ED is **conceptually lean** by design.
