# Foundations
### *The philosophical and conceptual grounding of the ED ontology*

The `foundations/` folder defines the **deep conceptual commitments** that make the ED ontology coherent.  
It is not physics, not mathematics, not PDEs, and not experiment‑driven.  
It is the **philosophical substrate** that explains *why* ED is built the way it is.

If the `interpretations/` folder explains *how to think in ED*,  
the `foundations/` folder explains *why ED is structured that way in the first place*.

This folder establishes the **first principles** of the ontology.

---

# Purpose of This Folder

The Foundations layer answers questions like:

- Why does ED begin with primitives instead of equations  
- Why is ED geometry‑first  
- Why are events the basic units of ontology  
- Why does ED rely on gradients, multiplicity, and commitments  
- Why is time relational rather than absolute  
- What philosophical lineage motivates ED  
- What conceptual pressures shaped the architecture  

These documents ensure that ED is:

- ontologically disciplined  
- structurally coherent  
- conceptually minimal  
- philosophically grounded  

They prevent hidden assumptions and keep the ontology clean.

---

This folder contains the following documents:

The philosophical lineage that motivates ED (structuralism, minimalism, relational ontology, etc.).

### **8. `conceptual_background.md`**  
The conceptual pressures and design constraints that shaped ED’s ontology.
