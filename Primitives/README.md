# primitives/ — How to Read the ED Primitives  
**A guide to the axioms of Event Density**

This folder contains the **13 primitives** of Event Density (ED).  
They are the **axioms** of the framework — the smallest, irreducible conceptual units from which all ED structure is built.

They are not derived.  
They are not justified by physics.  
They are not theorems.  
They are **choices** that define the ontology.

Everything else in ED — the PDE, quantum behavior, GR behavior, bridges, predictions — is built *on top of* these primitives, not inside this repo.

This folder is intentionally minimal and stable.

---

## What a “primitive” is in ED

A primitive is:

- **irreducible** — it cannot be defined in terms of other ED concepts  
- **structural** — it defines what exists, not how it behaves  
- **non‑numerical** — no equations, no parameters  
- **non‑physical** — no reference to experiments or platforms  
- **compositional** — it participates in building higher‑level structures  
- **stable** — it changes only after a deliberate tightening pass  

Each primitive is a *conceptual atom* of the ED ontology.

---

## How to read each primitive file

Each file in this folder (e.g., `01_micro_event.md`, `02_chain.md`) follows the same pattern:

### **1. Name**  
A short, stable label for the primitive.

### **2. Definition**  
A precise, minimal description of what the primitive *is*.  
This is the only part that is “constitutional.”

### **3. Role**  
What the primitive contributes to the ontology.  
This is conceptual, not physical.

### **4. Structural implications**  
How the primitive participates in building higher‑level ED structures.  
No equations, no physics — just ontology.

### **5. Notes / clarifications**  
Optional.  
Used to prevent misinterpretation or category errors.

---

## What *not* to expect in these files

The primitives do **not** contain:

- equations  
- PDEs  
- quantum mechanics  
- general relativity  
- platform bridges  
- experiments  
- predictions  
- retrodictions  
- references to specific physical systems  
- interpretations of news articles  
- diagrams of physical behavior  

Those belong in the **main Event Density repo**, not here.

---

## Why the primitives are numbered

The numbering (01–13) is:

- **stable** — numbers never change  
- **non‑hierarchical** — lower numbers are not more fundamental  
- **organizational** — helps with referencing and ordering  
- **version‑safe** — tightening passes do not reorder primitives  

The numbers are labels, not ranks.

---

## How the primitives relate to the rest of ED

The primitives define **what exists**.  
The main ED repo defines **how it behaves**.

You can think of it like this:

- `ED-primitives` = **ontology**  
- main ED repo = **theory + dynamics + applications**

This separation keeps the ontology clean and prevents conceptual drift.

---

## When primitives change

Changes to this folder are **rare** and treated as constitutional amendments.

A primitive may change only when:

- a conceptual inconsistency is discovered  
- a tightening pass clarifies the definition  
- the architecture demands a refinement  
- the primitive set becomes redundant or incomplete  

Changes must be:

- minimal  
- justified  
- documented  
- reviewed against the entire ontology  

This ensures stability across the entire ED program.
